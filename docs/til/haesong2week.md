## 1. ✨ 구현 기능

* **JWT 이중 토큰 체제 고도화 및 재발급 API (`/refresh`) 도입**
    * 기존의 단일 토큰 방식에서 탈피하여 보안성을 높인 **Access Token(30분)**과 **Refresh Token(7일)** 구조를 완전히 안착시킴
    * 토큰이 만료되어도 세션이 끊기지 않고 뒷단에서 유기적으로 연장되도록 검증 및 대조 비즈니스 로직을 구현
* **Swagger Security 단일 표준화 및 자동 Bearer 패치**
    * 자물쇠 설정 시 매번 귀찮게 `Bearer`를 수동으로 입력하던 불편함을 완벽히 해결
    * 순수 토큰값만 입력해도 스웨거가 자동으로 접두사를 붙여 HTTP 헤더에 실어 나르도록 설정을 표준화
* **`@RestControllerAdvice` 기반 전역 에러 관제탑 세팅**
    * 각 컨트롤러 내부를 지저분하게 채우고 있던 중복 `try-catch` 블록을 과감하게 싹 다 청소
    * 예외 발생 시 전역 관제탑이 에러를 가로채어 일관된 JSON 포맷(`ErrorResponse`)으로 응답하도록 예외 처리를 정돈

## 3. 💡 새롭게 알게 된 점

* **이중 토큰 보안 흐름과 Silent Refresh 메커니즘**
    * 실제 프론트엔드가 붙었을 때 사용자가 로그아웃을 체감하지 못하게 뒤에서 자동으로 토큰을 교체해 주는 **소리 없는 재발급(Silent Refresh)**의 전체적인 아키텍처 흐름을 파악함
* **Swagger OpenAPI 구성별 HTTP 헤더 제어권 차이**
    * `APIKEY` 타입은 원본 문자열을 그대로 전송하고, `HTTP Bearer` 타입은 스웨거가 직접 접두사를 조립한다는 메커니즘적 차이를 이해하고 올바른 빈(Bean) 설정을 적용함
* **통합 예외 처리의 생산성 향상**
    * 에러 관제탑을 세워두면 비즈니스 로직(Service)에서 단순 `throw`만 던져도 예쁜 규격으로 자동 포장되므로 코드 가독성이 기하급수적으로 올라간다는 것을 체감함

## ⚡ 어려웠던 점 (트러블슈팅)

### 🔍 문제 상황 1: 스웨거 자물쇠 인증 후 API 호출 시 지속적인 401 Unauthorized 발생
* **원인 분석**
    * 백엔드 코드는 헤더에서 `Bearer `(7글자)를 잘라내도록 설계되었으나, 기존 스웨거 설정이 `APIKEY` 방식으로 유지되어 있어 자물쇠 창에 순수 토큰만 넣었을 때 접두사가 누락된 채 전송되고 있었음
* **최종 해결**
    * `SwaggerConfig` 설정을 `Type.HTTP` 및 `.scheme("bearer")` 정석 규격으로 덮어써서 스웨거가 자동으로 `Bearer`를 조립해 쏘도록 처리하여 해결함

### 🔍 문제 상황 2: 코드 병합 및 리팩토링 후 unnamed classes 컴파일 실패 에러 발생
* **원인 분석**
    * 새 재발급 메서드를 파일 밑에 붙여넣는 과정에서 중괄호(`}`) 닫기 실수가 발생하여 메서드가 클래스 범위 바깥으로 탈출했고, 컴파일러가 이를 이름 없는 프리뷰 클래스로 오인함
* **최종 해결**
    * 파일 맨 마지막 행에 오직 클래스를 닫는 최종 중괄호 하나만 위치하도록 괄호 구조를 완전히 재배치하여 빌드를 정상화(`BUILD SUCCESSFUL`)함

## 📢 회의 때 설명할 내용 (3줄 요약)

1. 🔐 **이중 토큰 및 자동 연장 기반 완성:** Access/Refresh 분리 및 토큰 재발급 창구(`/refresh`)를 열어두었으니, 프론트엔드 담당자분은 Access 토큰 만료(401 에러) 감지 시 이 창구로 자동 연장 로직을 연결해 주시면 됩니다.
2. 🧼 **컨트롤러 소스코드 대청소:** 전역 에러 관제탑(`@RestControllerAdvice`)을 도입하여 컨트롤러마다 도배되어 있던 지저분한 `try-catch`를 싹 다 지우고 비즈니스 로직만 깔끔하게 남겨두었습니다.
3. 🔓 **스웨거 자물쇠 편의성 패치:** 이제 스웨거 테스트 시 자물쇠 창에 귀찮게 `Bearer`라고 손수 타이핑할 필요 없이, 로그인 성공 시 나오는 순수 토큰값만 복사·붙여넣기 하시면 즉시 모든 보안 API 호출이 가능합니다.
