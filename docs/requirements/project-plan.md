# 주식 시뮬레이터 기획서

## 1. 프로젝트 개요

### 프로젝트명

Stock Simulator

### 한 줄 소개

가상의 자산으로 주식을 사고팔며, 30분마다 변동되는 주가와 자산 랭킹을 경쟁하는 실시간 주식 시뮬레이션 웹게임입니다.

### 기획 배경

주식 투자는 많은 사람이 관심을 가지는 주제지만, 실제 돈을 사용하기 때문에 초보자가 쉽게 경험하기 어렵습니다. 이 프로젝트는 실제 투자와 비슷한 흐름을 게임처럼 가볍게 체험할 수 있도록 설계합니다.

사용자는 가상 현금을 지급받고, 9개의 가상 종목을 매수/매도하며, 시간이 지나면서 변동되는 주가에 따라 자산이 달라지는 경험을 하게 됩니다. 또한 실시간 채팅과 자산 랭킹을 통해 혼자 하는 시뮬레이션이 아니라 다른 유저와 함께 경쟁하는 게임성을 제공합니다.

## 2. 프로젝트 목표

### 핵심 목표

- 가상 주식 거래를 통해 투자 흐름을 직관적으로 경험할 수 있게 합니다.
- 30분 단위 주가 변동을 통해 실시간 게임처럼 즐길 수 있게 합니다.
- 자산 랭킹으로 경쟁 요소를 제공합니다.
- 실시간 채팅으로 유저 간 상호작용을 제공합니다.

### MVP 목표

- 회원가입/로그인 후 서비스를 이용할 수 있습니다.
- 유저는 9개 종목 중 원하는 종목을 매수/매도할 수 있습니다.
- 종목 가격은 30분마다 변동됩니다.
- 메인페이지에서 차트와 거래 기능을 사용할 수 있습니다.
- 마이페이지에서 내 자산과 보유 주식을 확인할 수 있습니다.
- 랭킹 페이지에서 전체 유저의 총 자산 순위를 확인할 수 있습니다.
- 메인페이지에서 실시간 채팅을 사용할 수 있습니다.

## 3. 타겟 사용자

### 주요 사용자

- 주식에 관심은 있지만 실제 투자는 부담스러운 사용자
- 투자 시뮬레이션을 게임처럼 즐기고 싶은 사용자
- 짧은 사이드 프로젝트로 실시간 기능, 차트, 거래 로직을 구현해보고 싶은 개발 팀

### 사용자 니즈

- 내 자산이 어떻게 변하는지 쉽게 확인하고 싶다.
- 주가가 오르고 내리는 흐름을 차트로 보고 싶다.
- 간단한 클릭과 입력으로 매수/매도를 하고 싶다.
- 다른 유저와 순위를 비교하고 싶다.
- 다른 유저와 실시간으로 이야기하고 싶다.

## 4. 핵심 게임 루프

1. 유저가 회원가입 또는 로그인합니다.
2. 초기 가상 현금을 지급받습니다.
3. 메인페이지에서 종목 목록과 차트를 확인합니다.
4. 가격 흐름을 보고 원하는 종목을 매수합니다.
5. 30분마다 주가가 변동됩니다.
6. 유저는 보유 종목을 추가 매수하거나 매도합니다.
7. 현금과 보유 주식 평가액을 합산한 총 자산이 변경됩니다.
8. 총 자산 기준으로 랭킹이 갱신됩니다.

## 5. MVP 기능 범위

### 5.1 회원 기능

#### 회원가입

- 사용자는 이메일, 닉네임, 비밀번호를 입력해 가입합니다.
- 이메일과 닉네임은 중복될 수 없습니다.
- 가입 시 초기 가상 현금이 지급됩니다.

#### 로그인

- 사용자는 이메일과 비밀번호로 로그인합니다.
- 로그인한 사용자만 거래, 채팅, 마이페이지 접근이 가능합니다.

#### 로그아웃

- 사용자는 현재 로그인 세션을 종료할 수 있습니다.

### 5.2 주식 기능

#### 종목 목록

- MVP 기준 총 9개 종목을 제공합니다.
- 각 종목은 이름, 코드, 현재가, 변동성 정보를 가집니다.

#### 주가 변동

- 주가는 30분마다 서버에서 자동 갱신됩니다.
- 종목별 변동성에 따라 가격 변경 폭이 다릅니다.
- 가격이 변경될 때마다 히스토리를 저장합니다.

#### 차트

- 선택한 종목의 가격 히스토리를 차트로 표시합니다.
- MVP에서는 30분 단위 가격 데이터를 기준으로 표시합니다.

### 5.3 거래 기능

#### 매수

- 유저는 종목과 수량을 선택해 매수할 수 있습니다.
- 보유 현금이 부족하면 매수할 수 없습니다.
- 매수 성공 시 현금이 차감되고 포트폴리오 수량이 증가합니다.
- 평균 매수가를 갱신합니다.

#### 매도

- 유저는 보유 중인 종목과 수량을 선택해 매도할 수 있습니다.
- 보유 수량보다 많은 수량은 매도할 수 없습니다.
- 매도 성공 시 포트폴리오 수량이 감소하고 현금이 증가합니다.

#### 거래 내역

- 유저의 매수/매도 기록을 저장합니다.
- MVP에서는 마이페이지 또는 별도 영역에서 최근 거래 내역을 표시할 수 있습니다.

### 5.4 마이페이지

- 내 닉네임
- 보유 현금
- 보유 종목 목록
- 종목별 보유 수량
- 종목별 평균 매수가
- 종목별 현재 평가 금액
- 총 자산
- 전체 수익률

### 5.5 랭킹

- 모든 유저의 총 자산을 기준으로 순위를 계산합니다.
- 총 자산은 보유 현금과 주식 평가 금액의 합입니다.
- 랭킹 페이지에서 순위, 닉네임, 총 자산, 수익률을 표시합니다.

### 5.6 실시간 채팅

- 로그인한 유저는 메인페이지에서 채팅을 사용할 수 있습니다.
- 메시지는 닉네임, 내용, 작성 시간과 함께 표시됩니다.
- WebSocket을 사용해 실시간으로 메시지를 송수신합니다.

## 9. 정책 및 예외 처리

### 회원 정책

- 이메일은 중복 가입할 수 없습니다.
- 닉네임은 중복 사용할 수 없습니다.
- 비밀번호는 암호화해서 저장합니다.

### 거래 정책

- 매수 금액이 보유 현금보다 크면 거래가 실패합니다.
- 매도 수량이 보유 수량보다 크면 거래가 실패합니다.
- 수량은 1주 이상만 입력할 수 있습니다.
- 거래 시점의 현재가를 기준으로 체결합니다.

### 주가 정책

- 주가는 0 이하가 될 수 없습니다.
- 주가 변경은 서버 기준 시간으로 처리합니다.
- 가격 히스토리는 차트 표시를 위해 저장합니다.

### 채팅 정책

- 로그인한 유저만 채팅할 수 있습니다.
- 빈 메시지는 전송할 수 없습니다.
- MVP에서는 욕설 필터링과 신고 기능은 제외합니다.

## 14. 리스크 및 고려사항

### 실시간 처리 범위

주가 변경, 차트 갱신, 채팅이 모두 실시간 요소를 가지므로 MVP에서는 복잡도를 줄이기 위해 주가는 30분 단위 API 갱신으로 처리하고, 채팅만 WebSocket을 우선 적용합니다.

### 랭킹 계산 비용

유저 수가 많아질 경우 모든 유저의 보유 주식을 매번 계산하는 비용이 커질 수 있습니다. MVP에서는 요청 시 계산하거나 간단한 캐싱을 적용하고, 추후 별도 랭킹 테이블이나 배치 갱신을 고려합니다.

### 거래 동시성

동시에 여러 거래 요청이 들어오면 현금과 보유 수량 정합성이 깨질 수 있습니다. 백엔드에서는 트랜잭션을 적용하고, 필요하면 유저 자산 또는 포트폴리오 단위의 잠금 전략을 검토합니다.
